为什么不使用宏定义

发表于 2021-04-24
宏定义的缺陷
  1. 宏定义在预处理阶段被替换，不会进入符号表，会给调试与处理编译错误增加困难。
  2. 宏定义在所有出现的地方进行替换，将增大生成的目标代码的体积。
  3. 宏定义无法限制作用域，更不能提供任何封装性。
  4. 宏定义难以避免调用时的边际效应（诸如自增等操作符可能使宏定义不合预期）。
正确的编程实践
  1. 使用 const，enum 等定义常量；
  2. 使用 inline 函数。
